主要フォルダの詳細説明

__tests__ DIR

各種テストコード（主に unit 内のユニットテスト）を格納するフォルダ。

docs DIR

仕様書やプロジェクト構造説明書などの関連ドキュメントを配置するフォルダ。現在は空です。

events DIR

sam local invoke コマンド等でLambda関数をローカル実行する際に使用する、API Gateway等の疑似リクエストイベント（JSONファイル）が格納されています。

src DIR

バックエンドアプリケーションのメインソースコード群です。クリーンアーキテクチャやDDD（ドメイン駆動設計）の概念に基づいたレイヤー構成を採用しています：

application/: ユースケース（CreateChatPointUseCase 等）など、ビジネスロジックを実行するアプリケーション層。
domain/: ドメインエンティティ（pointInfo.ts など）や値オブジェクトなどのコアロジックを配置するドメイン層。
handlers/: Lambdaハンドラの共通定義などのエントリーポイント関連コード。
infrastructure/: AWS DynamoDB（DynamoMapRepository 等）や Firebase、GCP、Cognito などの外部サービスとのやり取りを担う具象実装クラスを格納するインフラストラクチャ層。
interfaces/: AWS API Gatewayの各エンドポイントやSQSトリガーに対応する、Lambdaエントリーハンドラ（interfaces/handlers/）などを配置するインターフェース層。
shared/: ログ出力（logger.ts）やMIMEタイプ変換ヘルパーなど、プロジェクト全体で共有するユーティリティ。
tests DIR

テスト用の共通構成。Jest実行前に実行される setup.ts などが配置されています。

設定ファイル・ルートドキュメント

.eslintignore / .eslintrc.js FILE

JavaScript/TypeScriptの静的コード解析ツール ESLint の設定および解析対象外指定ファイル。

.gitignore FILE

Gitリポジトリ管理から除外するディレクトリおよびファイルを定義します。

.npmignore FILE

npmパッケージ化やデプロイ時に含めないファイルを設定するファイル。

.prettierrc.js FILE

コードフォーマッタ Prettier のスタイルルール設定ファイル。

LICENSE FILE

プロジェクトのソフトウェアライセンス（MIT License）が明記されたファイル。

README.md FILE

プロジェクトの概要、構成、デプロイやテストの方法などを記述した基本ドキュメント。

buildspec.yml FILE

AWS CodeBuildがビルドやデプロイ時に使用する処理手順（フェーズごとのコマンド）を定義したファイル。

env.json FILE

ローカル実行時にLambda関数へ渡す環境変数を定義するJSONファイル。

jest.config.js FILE

テストフレームワーク Jest の設定ファイル。tsconfig.test.json を用いた TypeScript のトランスパイルやカバレッジ取得の設定が含まれます。

package.json FILE

依存ライブラリや開発用ライブラリ、およびビルド・テスト・デプロイ用スクリプト（npm run compile, npm run build, npm run deploy-poc 等）を管理するファイル。

samconfig.toml FILE

AWS SAM CLIで使用するデフォルトパラメータ（スタック名、S3バケット等のデプロイ設定）を記述する設定ファイル。

swagger.yaml FILE

OpenAPI 3.0 規格に基づき、提供するWeb APIのエンドポイントやリクエスト/レスポンススキーマを定義したAPI仕様書ファイル。

template.yaml FILE

AWS SAM（Serverless Application Model）仕様に基づき、Lambda関数、DynamoDBテーブル、Cognitoユーザープール、S3バケット、SQSキューなどのAWSリソースおよびそれらの権限（IAM）関係を定義するインフラテンプレート（CloudFormationの拡張）ファイル。

tsconfig.json / tsconfig.test.json FILE

TypeScriptのコンパイルオプション（ビルド用およびテストコード実行用）を定義するファイル。

Git除外対象（.gitignore）

機密情報の流出防止や、ビルド生成ファイルによる容量肥大化を避けるため、以下のリソースは Git 管理から明示的に除外されています。

node_modules/
npm パッケージ依存関係の実体。
.aws-sam/
AWS SAMのビルド結果やキャッシュ、パッケージング用ファイル。
dist/
TypeScriptのコンパイル出力ディレクトリ。
secret/
Firebaseサービスアカウントの秘密鍵など、機密情報フォルダ。
.env*
検証・ローカル用の機密環境変数ファイル。
jest.config.js.map
ビルドにより副次生成されるテスト用のソースマップファイル。
test.md / *.json / *.ps1
一時的なスクリプトやポリシー定義、テスト用マークダウンなど。
.git/
Git の変更履歴・管理メタデータ。
